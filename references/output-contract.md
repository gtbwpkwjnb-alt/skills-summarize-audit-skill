# 审计输出与事实契约

审计的第一轮只能读取和分析。报告中的每个结论必须带状态和证据，不得用样例数字、缓存缺失时的默认高分或模型常识补齐事实。

## 事实状态

| 状态 | 含义 | 输出要求 |
|:---|:---|:---|
| `observed` | 当前扫描、命令或可复查文件直接得到 | 给出路径、命令或记录时间 |
| `inferred` | 从观察到的数据按规则推导 | 给出规则和输入证据 |
| `estimated` | 使用近似公式计算 | 给出公式、输入与误差说明 |
| `unavailable` | 数据未读取、来源不可信或功能未启用 | 明确说明缺口，不参与排序或综合分 |

## 输出规则

1. 健康度、趋势、活性、收益、容量和推荐必须逐项标明事实状态。
2. 外部信号关闭时，Community 为 `unavailable`；本地 registry 只能说明“registry 对齐”，不能证明版本新鲜度。
3. 结构化 Codex session/tool-call 事件缺失或无法可靠归因时，使用证据为 `unavailable`；不得从旧版 error ledger、系统提示、skill catalog 或原始 prompt 回填分数或触发质量缺口。
4. 容量只输出 `estimated`，并注明“文件字节/4 不是实际注入 token”。
5. 没有证据 URL、许可证、兼容性或作用域理由的候选只能标为“评估中”。
6. 未确认前，所有操作只能标记为 `[需确认]`，不能创建画像、缓存、日志、快照或修改目标文件。

## 报告最小结构

```text
审计状态: complete | partial
证据覆盖: observed N / inferred N / estimated N / unavailable N
关键结论: 每项包含状态、证据和限制
操作建议: 仅 [需确认] 项，含作用域、目标路径、风险与回滚方式
未获取数据: 明确列出，不用默认值替代
```

`report_status=complete` 只表示已完成既定扫描，不表示所有维度都有数据；存在 `unavailable` 时必须保留该清单。
