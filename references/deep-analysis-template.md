# Deep Analysis Template — Skills Audit v5.2.0

> 逐技能六维分析 + 维护清单 + 参考验证的完整输出格式。参见 SKILL.md 步骤④-bis。

## 六维分析模板

```text
📋 {技能名} v{版本} — {层级}

  作用: {这个技能在你当前项目中具体解决什么问题？基于正文内容描述，不靠 description 猜}

  适配:
    ✅ {从正文中发现的与项目活动契合的具体功能点}
    ❌ {从正文中发现的不适配点——硬编码路径、缺失依赖、功能重叠}

  缺口:
    {技能正文中引用了但经验证不存在的文件/路径/依赖}
    {技能正文提到但未实现的功能}

  建议:
    {具体可操作的优化方向，含优先级和预期效果}
```

**判定规则**：
- "作用" 必须锚定项目画像的核心活动
- "适配" 的 ✅/❌ 必须引用技能正文具体行/段作为证据
- "缺口" 必须经工具验证（ls/Test-Path/grep 确认文件不存在）
- "建议" 每条 ≤3 句话，含 `🔍 原因` `💡 效果` `🎯 场景`

## 维护清单模板

```text
维护:
  🔄 更新: {版本 vs registry 最新 → 是否需要更新}
  ✅ 环境: {依赖命令/文件是否存在 → pass/fail}
  📝 描述: {description 是否合规 → ok/需优化}
  ⚠️ 注意: {安全/兼容性/许可证等注意事项}
```

检查规则：
- `🔄 更新` — 对比 installed 版本 vs `skill-registry.yaml` 记录
- `✅ 环境` — 验证引用的外部命令/路径是否存在
- `📝 描述` — 复用步骤④的描述质量检查结果
- `⚠️ 注意` — 从技能正文提取安全注意事项

## 参考验证模板

```text
参考验证:
  🔍 {技术/依赖名} → 来源: {GitHub/PyPI/npm} · 版本: {latest} · 社区活跃: {活跃/一般/停滞}
  📄 参考文档: {URL}
```

> 必须使用 `agent-reach` 或 `web_fetch` 验证至少 2 个独立来源。
> 网络不可用时标注「⏸ 离线，建议网络恢复后验证」。
