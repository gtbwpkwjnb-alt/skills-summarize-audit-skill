# 执行流程

只在用户请求的能力范围内执行，不串联为技能总管。

```text
定位当前 UI 可见性证据 → 执行所请求的核心能力之一或组合 → 输出只读结论与下一步建议
```

| 能力 | 输入 | 读取资源 | 输出 | 明确不做 |
|---|---|---|---|---|
| 技能/插件问题审查 | 已安装清单、可见 ID 或用户画像 | `audit_skill_plugin_issues.py`、`user-profile.md`、健康/冲突/推荐规则 | 来源/版本/可用性问题、健康分、画像匹配、冲突/互补和处理方案 | 安装、更新、修改 cache/system skill 或推送 |
| 翻译精炼 | 当前侧栏/命令栏截图、技能 ID 列表或已验收导出 | `display-source-map.md`、术语库、`--scope visible` 采集器 | 仅可见项的中文触发词、简介、来源、质量、可编辑性 | 推断 UI 可见性、扫描未显示项、写 UI/cache/系统技能/插件 |
| 项目画像 | 当前/指定项目目录 | `project-types.yaml`、项目文件与 Agent 规则 | 技术栈、工作流、约束、能力缺口 | 写配置、修复环境或迁移桌面数据 |
| 推荐 | 翻译清单、项目画像或明确缺口 | 本地清单；联网同意后才查外部证据 | 保留/观察/考虑引入/不建议及理由 | 安装、更新、卸载、发布或执行变更 |

报告模板见 `report-template.md`。需要实施时，报告只写明应直接调用的 Agent 类型、目标和所需输入。
