# 回滚流程测试 — 快照备份 + --undo

> 验证 ⑦-a 快照备份 + ⑦-d 回滚操作

## 前置条件

```yaml
snapshot:
  enabled: true
  dir: "~/.agents/skills/.audit-snapshots/"
  retention_days: 7
  auto_before_exec: true
```

## 用例 1：执行前自动创建快照

| 步骤 | 操作 | 预期结果 |
|:----|:-----|:---------|
| ① | 触发 `技能审查`，选择执行归档操作 | ✅ |
| ② | ⑦-a 执行前自动备份 | `~/.agents/skills/.audit-snapshots/20260705T100000/` 创建 |
| ③ | manifest.json 生成 | 含 timestamp, skill_list, file_count, total_size |
| ④ | 摘要输出 | `📸 快照已创建 (3 个技能, 12KB) — 使用 '技能审查 --undo 20260705T100000' 恢复` |

## 用例 2：回滚操作

| 步骤 | 操作 | 预期结果 |
|:----|:-----|:---------|
| ① | 触发 `技能审查 --undo 20260705T100000` | ✅ |
| ② | 查找快照目录 | `{dir}/20260705T100000/` 存在 |
| ③ | 展示变更清单 | 对比快照与当前状态，列出差异文件 |
| ④ | 用户确认 | 等待 y/N |
| ⑤ | 用户确认后恢复 | 快照文件复制回原路径 |
| ⑥ | .data/stats.json 标记 | `rolled_back: true`, `snapshot_id: 20260705T100000` |

## 用例 3：列出可用快照

| 步骤 | 操作 | 预期结果 |
|:----|:-----|:---------|
| ① | 触发 `技能审查 --undo --list` | ✅ |
| ② | 输出 | 按时间倒序列表，含 timestamp, skill_count, total_size |

## 用例 4：快照过期清理

| 输入 | 预期输出 |
|:----|:---------|
| 快照目录已有 8 天前的旧快照 | ⑧ 持久化阶段自动清理 |
| retention_days: 7 | 旧快照被删除 |
| 清理失败 | 不阻塞主流程，仅输出警告 |
