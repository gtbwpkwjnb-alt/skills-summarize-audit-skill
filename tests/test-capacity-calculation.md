# 容量计算测试 — 多因子有效容量模型

> 验证 ②-bis 多因子有效容量计算公式

## 计算公式

```
有效容量 = min(
  token预算容量,
  技能上下文容量 × 冲突因子,
  认知容量
)
冲突因子 = 1 - (contamination均分 / 5 × 0.3)
认知容量 = max(3, min(15, activity_count × 2))
```

## 用例 1：CodeBuddy 正常场景

| 参数 | 值 |
|:----|:----|
| Agent | CodeBuddy (16K) |
| SystemPrompt + 历史 + 输入 | 6300t |
| 单 skill 平均成本 | 3000t |
| token预算容量 | (16000 - 6300) / 3000 = 3.2 |
| contamination均分 | 2.0 |
| 冲突因子 | 1 - (2.0/5 × 0.3) = 0.88 |
| 技能上下文容量 | 3.2 × 0.88 = 2.8 |
| activity_count | 3 |
| 认知容量 | max(3, min(15, 6)) = 6 |
| **有效容量** | **min(3.2, 2.8, 6) = 2.8 → 🟡 可再装 2-4** |
| 已安装 | 11 |
| 可再装 | 2.8 - 11 = 负数 → **🔴 已满** |

## 用例 2：Claude Code 大窗口场景

| 参数 | 值 |
|:----|:----|
| Agent | Claude Code (200K, listingBudget=2000t) |
| 固定消耗 | 12000t |
| token预算容量 | (200000 - 12000) / 3000 = 62.7 |
| contamination均分 | 1.0（低重叠） |
| 冲突因子 | 1 - (1.0/5 × 0.3) = 0.94 |
| 技能上下文容量 | 62.7 × 0.94 = 58.9 |
| activity_count | 5 |
| 认知容量 | max(3, min(15, 10)) = 10 |
| **有效容量** | **min(62.7, 58.9, 10) = 10** |
| 已安装 | 8 |
| 可再装 | 10 - 8 = **2 → 🟡 注意** |

## 用例 3：高重叠污染扣分场景

| 参数 | 值 |
|:----|:----|
| contamination均分 | 4.5（严重重叠） |
| 冲突因子 | 1 - (4.5/5 × 0.3) = 0.73 |
| 有效容量 = token预算容量=5, 技能上下文=5×0.73=3.65, 认知=8 | **min(5, 3.65, 8) = 3.65 → 🟡** |
| 对比无冲突因子（5 → 🟢） | 冲突因子让判定从 🟢→🟡 **更准确** ✅ |
